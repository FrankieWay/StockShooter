# 短线热门股挖掘

从核心新闻站抓取近期热点 → 提取关键信息并映射到 A 股产业 → 在产业内筛选高潜力、筹码密集、易爆发个股 → 按爆发置信度从高到低输出清单。

## 工作流总览

按顺序执行以下四步，每步完成后再进入下一步：

```
进度清单：
- [ ] 步骤 1：新闻抓取与热度排序
- [ ] 步骤 2：关键信息提取 + 产业映射
- [ ] 步骤 3：产业内个股筛选（高潜力 + 筹码密集）
- [ ] 步骤 4：爆发置信度打分并输出清单
```

---

## 步骤 1：新闻抓取与热度排序

### 时间范围
- 仅使用 **最近 3～5 天** 的新闻。

### 数据来源（优先使用可爬取或可搜索的站点）
- **国内**：财联社、东方财富快讯/要闻、新浪财经、新华网财经、人民网财经、证券时报、第一财经、同花顺要闻。
- **国际/综合**：华尔街见闻、金十数据、Reuters/ Bloomberg 中文摘要（若有）。

使用 `web_search` 或 `mcp_web_fetch` 获取标题与摘要；若用户环境有爬虫脚本，可执行脚本批量抓取。禁止伪造新闻，未抓到的来源可注明“未获取”。

### 热度排序规则
1. **时效**：越近的新闻权重越高（如 24h > 48h > 72h）。
2. **来源权威性**：央媒、头部财经媒体权重高于一般站点。
3. **主题**：世界经济、中国宏观、产业政策、重大产品/技术、地缘政治优先。
4. 输出：**热点新闻列表**（标题、来源、日期、一句话摘要），按综合热度从高到低排序，数量控制在 15～30 条为宜。

---

## 步骤 2：关键信息提取与产业映射

### 关键信息提取
从步骤 1 的每条新闻中提取：
- 核心事件（政策名、技术名、产品名、地区/国家、公司/行业名）。
- 明确受益或受损方向（如“新能源车”“半导体”“消费”“出口”等）。

### 产业映射
将上述关键词映射到 **A 股申万行业或常见板块**（见 [reference.md](reference.md)）。  
规则：
- 一个热点可对应多个产业；标出主产业与次产业。
- 仅保留与 A 股有明确关联的产业（有对应上市公司）。
- 输出：**热点 → 产业映射表**（热点摘要、主产业、次产业、关联强度：强/中/弱）。

---

## 步骤 3：产业内个股筛选

在步骤 2 得到的每个**主产业**内，筛选同时满足以下特征的个股：

### 高潜力
- 与当前热点逻辑直接相关（主营或主要收入来自该产业）。
- 近期有催化剂（公告、研报、订单、政策落地等）更佳。

### 筹码密集、易爆发
- **筹码集中**：股东户数近期下降或户均持股上升；或前十大股东/流通股东持股比例较高且稳定。
- **流动性适中**：日成交额不宜过小（如 ≥ 5000 万），避免无法进出。
- **技术位**：优先考虑整理后放量、突破关键均线或前高的标的（若无法获取 K 线，可跳过此项，在置信度中注明“未校验技术面”）。

数据来源：东方财富、同花顺、交易所公告、年报/季报。可用 `web_search` 查“某行业 筹码集中 股东户数”等。**若用户提供 Tushare token**，可优先用 Tushare 获取股东户数、户均持股、日成交额及行业分类（见 [reference.md](reference.md) 第四节），数据更结构化、便于筛选。

### 输出
每个产业给出 **3～8 只** 候选标的，并简要注明：代码、名称、与热点的关联、筹码/流动性要点。

## 步骤 4：爆发置信度打分与清单输出

### 置信度维度（可加权综合）
| 维度 | 说明 | 高分条件 |
|------|------|----------|
| 热点相关度 | 新闻与公司业务的匹配度 | 直接受益、逻辑清晰 |
| 筹码与资金 | 筹码集中 + 成交温和放量 | 户数降、户均持股升、近期有放量 |
| 催化剂 | 公告/政策/订单时间远近 | 近期有明确催化剂 |
| 流动性 | 成交额与流通盘 | 成交额适中、能进出 |

### 打分与排序
- 为每只候选股在以上维度打分（如 1～5 分），再汇总为 **爆发置信度**（可简单加权平均或自定义权重）。
- 按 **爆发置信度从高到低** 排序，得到最终 **个股清单**。

### 必须使用的输出模板

使用以下 Markdown 模板输出，不得省略“免责声明”：

```markdown

**数据基准日**：YYYY-MM-DD  
**热点时间窗口**：最近 N 天

### 一、热点新闻与产业映射摘要
（步骤 1～2 的浓缩：前 5～10 条热点 + 对应产业）

### 二、个股清单

| 排名 | 代码 | 名称 | 所属产业 | 爆发置信度 | 核心逻辑简述 |
|------|------|------|----------|------------|--------------|
| 1    |      |      |          | 高/中高/中  |              |
| 2    |      |      |          |            |              |
| …    |      |      |          |            |              |

### 三、重点标的简要说明（前 3～5 只）
对置信度最高的 3～5 只分别写 1～2 句：热点关联 + 筹码/催化剂要点。

### 四、数据与免责声明
- 新闻与数据来源：（列出本报告实际使用的来源）
- 本清单仅供研究参考，不构成任何投资建议；股市有风险，投资需谨慎。
```

## 注意事项

- **不编造新闻或数据**：未抓取到的来源明确标注；无法获取的指标在清单中注明“未验证”。
- **统一使用 A 股**：港股/美股若提及，需单独区分，清单主体为 A 股。
- **路径**：脚本或链接一律使用正斜杠，例如 `scripts/fetch_news.py`。
- 若用户提供自有数据源或 API，优先按用户指定来源执行，并在最终清单中注明数据来源。
- **Tushare**：非必需。若项目根目录 `.env` 中存在 `TUSHARE_TOKEN` 或用户提供了 token，则用于步骤 3 的筹码与行情数据（见 reference.md）；未配置时仅用网页搜索与公开数据。
